---
title: "POST /products/recrawl - schedule product crawls"
description: "POST /products/recrawl schedules one or more product URLs or UUIDs for an on-demand crawl across your granted catalogs."
sidebarTitle: "Recrawl Products"
---

`POST /products/recrawl` schedules specific products for an on-demand crawl. Use it when you need Octogen to refresh a known set of product pages sooner than the daily sitemap-driven crawl. The endpoint returns as soon as crawler tasks are scheduled; it does not wait for crawling, extraction, enrichment, or indexing to finish.

## Request

```http
POST https://api.octogen.ai/v1/products/recrawl
Authorization: Bearer <your-platform-api-key>
Content-Type: application/json
```

### Body parameters

<ParamField body="targets" type="object[]" required>
  Product identifiers to schedule for recrawl. Maximum 500 targets per request.
</ParamField>

<ParamField body="targets[].url" type="string">
  Product page URL to recrawl. Provide either `url` or `uuid`, not both.
</ParamField>

<ParamField body="targets[].uuid" type="string">
  Octogen product UUID to recrawl. Provide either `uuid` or `url`, not both.
</ParamField>

<ParamField body="targets[].catalog" type="string">
  Optional catalog key. Include this when the URL is not already indexed, or when the same URL could belong to more than one granted catalog.
</ParamField>

### Example

```bash
curl -sS https://api.octogen.ai/v1/products/recrawl \
  -H "Authorization: Bearer $OCTOGEN_PLATFORM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "targets": [
      {"url": "https://warrenlotas.com/products/black-hoodie"},
      {"uuid": "11111111-1111-4111-8111-111111111111"},
      {
        "catalog": "warrenlotas",
        "url": "https://warrenlotas.com/products/new-drop"
      }
    ]
  }'
```

## Response

The endpoint returns `202 Accepted` when at least one target was scheduled. It returns `400 Bad Request` with the same response shape when no target could be scheduled.

<ResponseField name="requestId" type="string" required>
  Request identifier used to group the crawler tasks created for this call.
</ResponseField>

<ResponseField name="submitted" type="integer" required>
  Number of targets received in the request.
</ResponseField>

<ResponseField name="tasksCreated" type="integer" required>
  Number of crawler tasks created.
</ResponseField>

<ResponseField name="taskIds" type="string[]" required>
  Crawler task IDs created for accepted targets.
</ResponseField>

<ResponseField name="accepted" type="object[]" required>
  Targets accepted for crawling after authorization and validation.
</ResponseField>

<ResponseField name="rejected" type="object[]" required>
  Targets that were not scheduled. Each rejection includes the original `target`, a machine-readable `code`, and a human-readable `message`.
</ResponseField>

### Example response

```json
{
  "requestId": "62cf3df6-30f2-4c0e-b13d-fc509bba8d4f",
  "submitted": 3,
  "tasksCreated": 1,
  "taskIds": ["recrawl-62cf3df6-30f2-4c0e-b13d-fc509bba8d4f-warrenlotas-0001-products"],
  "accepted": [
    {
      "catalog": "warrenlotas",
      "url": "https://warrenlotas.com/products/black-hoodie"
    }
  ],
  "rejected": [
    {
      "target": {
        "url": "https://unknown.example/products/1"
      },
      "code": "product_not_found",
      "message": "No active product matched that URL in granted catalogs."
    }
  ]
}
```

## Resolution rules

| Target | Behavior |
| --- | --- |
| `{"uuid": "..."}` | Finds the product UUID within your granted catalogs and schedules its canonical `productUrl`. |
| `{"catalog": "...", "uuid": "..."}` | Finds the UUID only in the requested granted catalog. |
| `{"url": "..."}` | Finds the URL in your granted catalogs. If multiple catalogs match, the target is rejected with `ambiguous_url`. |
| `{"catalog": "...", "url": "..."}` | Schedules the URL for that granted catalog, even if the product is not indexed yet. |

## Errors

| Status | `detail` or rejection code | Meaning |
| --- | --- | --- |
| `400` | response body with `rejected` targets | No target could be scheduled. |
| `403` | `"catalog_not_granted"` | The request named a catalog your API key cannot access. |
| `422` | validation error array | The body is malformed, has more than 500 targets, or a target contains both `url` and `uuid`. |
| `503` | `"crawler_service_unavailable"` | Octogen could not contact the internal crawler scheduler. Retry later. |

<Note>
  A scheduled recrawl only starts the crawl pipeline. Downstream extraction,
  enrichment, indexing, and image processing still complete asynchronously.
</Note>
